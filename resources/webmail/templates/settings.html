Transfer Settings
When transfering email, messages have to be broken into lines. The standard specifies lines should not exceed 78 characters although most foriegn servers as well as all Chainmail servers can handle more. Setting this higher than 998 will, however, cause delevery to most foreign servers to fail.
If set to "No" Chainmail will not accept transfer of message data from any foreign server that's not using SSL. This forces users sending data to you to use SSL, which is more secure, but you run the risk of mail not getting though if the sender can't or won't use SSL.
